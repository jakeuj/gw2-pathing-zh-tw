# Contact templates

## GitHub issue

Title:

> License clarification / permission for a non-commercial Traditional Chinese (zh-TW) translation

Body template:

> Hi **[author]**,
>
> Thank you for creating and maintaining **[pack names]**.
>
> I am planning a non-commercial Traditional Chinese (`zh-TW`) localization for Blish HUD Pathing. Marker-pack content currently does not appear to support selecting localized strings at runtime, so the translated version would need to be distributed as separate `.taco` package(s) from a public GitHub project.
>
> The first version would only translate user-facing text, including category `displayname` values, marker `info` text, and user-facing `copy-message` text. I would not modify routes, coordinates, icons, images, scripts, or gameplay behavior. The translated package would, however, need to include the unchanged original files required for the pack to function.
>
> I would credit you prominently, link to the original source, identify the translated text and other changes, and keep the project free, without a paywall, and non-commercial.
>
> I could not find an explicit license covering modification and redistribution of the marker-pack content. Would you be willing to do either of the following?
>
> 1. Add a license or README statement of your choice that explicitly applies to the marker-pack content and the assets you have the right to license; or
> 2. Grant project-specific permission by replying with a statement such as the following (you are welcome to edit its conditions):
>
>    > I, **[author/handle]**, give **jakeuj** permission to create, maintain, and publicly redistribute non-commercial Traditional Chinese (`zh-TW`) versions of **[pack names]**, including unchanged original files needed for the translated packages to function. The translated versions must credit me, link to the original source, and identify the translated or modified text. Additional conditions: **[none/conditions]**. This permission covers **[the current version only/future versions I publish]** and excludes third-party content I do not have authority to license.
>
> Could you also let me know:
>
> - whether any bundled third-party content must be excluded;
> - whether permission applies only to the current version or also to future versions you publish;
> - your preferred attribution wording; and
> - whether you would prefer to publish the `zh-TW` version yourself instead?
>
> I will not publish a translated version without explicit permission. Thank you for considering the request.

## Fourteen-day follow-up

> Hi, just a gentle follow-up in case this request was missed. No action is required if you would rather not permit a translated redistribution; I will not publish one without explicit permission. Thank you again for your time.

> Thank you. To make sure I understand correctly, does your approval explicitly include both (1) translating the user-facing text and (2) publicly redistributing a separate non-commercial `.taco` package containing the unchanged files required for the pack to function? Please also note whether the approval covers future versions you publish and whether any third-party assets are excluded.
